Подготовка и запуск нейронной сети на Jetson nano

Подготовка и запуск нейронной сети на Jetson nano
§правка

Существует несколько подходов к оптимизации графа вычислений нейронных сетей с использованием TensorRT:
  1. Tensorflow -> uff format -> TensorRT
  2. Tensorflow_TensorRT (TF_TRT)
  3. Tensorflow -> onnx format -> TensorRT

Через uff format
§правка

Сама nvidia предоставляет примеры сетей для оценки работы оборудования именно в этом формате, однако nvidia заканчивает его поддержку.
Открываем nano и добавляем строки в конец файла
nano ~/.bashrc  
export PATH="/usr/local/cuda/bin:${PATH}"
export LD_LIBRARY_PATH="/usr/local/cuda/lib64:${LD_LIBRARY_PATH}"
export CPATH=$CPATH:/usr/local/cuda-10.2/targets/aarch64-linux/include
export LIBRARY_PATH=$LIBRARY_PATH:/usr/local/cuda-10.2/targets/aarch64-linux/lib

sudo apt-get install python3-pip
pip3 install pycuda --user
Конвертация в .uff:
При использовании стандартного инструмента конвертации возникают ошибки из-за невозможности представления определенных слоев используемой архитектуры в формате .uff
python3 /usr/lib/python3.6/dist-packages/uff/bin/convert_to_uff.py
Решением данной проблемы сдала возможность преобразовывать граф tensorflow с помощью graphsurgeon, позволяющем представить неподдерживаемые слои в поддерживаемом .uff формате. Для этого был написан скрипт custom_tf_uff/tf2uff.py:
python3 custom_tf_uff/tf2uff.py
Если предполагается его использование с другими параметрами сети необходимо изменить значения параметров внутри скрипта.
Для установки числа классов необходимо устанавливать значение:
numClasses=n+1 , где n– это фактическое число классов
При возникновении ошибок типа:
#assertionnmsPlugin.cpp,246
or
#assertionnmsPlugin.cpp,249
Неверно указан параметр inputOrder плагина NMS. Для этого в скрипте конвертации графа выполняем вызов uff.from_tensorflow с параметром text=1, после чего создается текстовый файл описания сети. В нем указана последовательность:
    id: "NMS"
    inputs: "Squeeze"
    inputs: "concat_priorbox"
    inputs: "concat_box_conf"
Требуемая последовательность для параметра inputOrder составляет {loc_data, conf_data, priorbox_data}. Следовательно:
inputOrder=[0, 2, 1],
Запуск сети:
Для запуска сети был написан скрипт custom_tf_uff/start_uff_trt.py
python3 custom_tf_uff/start_uff_trt.py

TF_TRT
§правка

git clone --recursive https://github.com/jkjung-avt/tf_trt_models
cd tf_trt_models/
./install.sh
Установка protobuf для увеличения скорости оптимизации сети (если необходимо):
wget https://raw.githubusercontent.com/jkjung-avt/jetson_nano/master/install_protobuf-3.6.1.sh
./install_protobuf-3.6.1.sh
Запуск работы сети на видео:
Для запуска скрипта необходимо добавить ссылку на свою сеть в файле /utils/egohands_models.py, указать ссылку на файл labels.pbtxt, указать место сохранения оптимизированного графа в переменную pb_path и настроить желаемый вывод прямого прохода сети в файле camera_tf_trt.py.
При первом запуске необходимо запустить процесс построения оптимизированного графа с помощью флага –build в команде запуска:
cd /home/varya/tf_trt_models/
python3 camera_tf_trt.py --file --filename /media/varya/SP\ UFD\ U2/Object_detection_api/create_dataset/videos/20200617_171311.avi --model model  --build

Через onnx format
§правка

!!! Пока НЕ УДАЛОСЬ применить данный подход, возникает ошибка на этапе построения trt engine.
Устанавливаем onnx:
git clone https://github.com/onnx/onnx.git
cd /home/varya/onnx/
git submodule update --init –recursive
sudo apt-get install protobuf-compiler
sudo apt-get install libprotoc-dev  
python3 setup.py install
Устанавливаем tensorflow-onnx:
git clone https://github.com/onnx/tensorflow-onnx
cd /home/varya/tensorflow-onnx/
python3 setup.py install
Конвертирование графа .pb в .onnx :
Важно! Установка –opset 11
Если из сохраненной модели папки saved_model:
python3 -m tf2onnx.convert --saved-model /media/varya/SP\ UFD\ U2/Object_detection_api/inf371847/saved_model --output frozen_ssd.onnx --opset 11
Из замороженного графа модели:
python3 -m tf2onnx.convert --graphdef /media/varya/SP\ UFD\ U2/Object_detection_api/inf371847/frozen_inference_graph.pb --output frozen_ssd.onnx --opset 11 --inputs image_tensor:0 --outputs num_detections:0,detection_classes:0,detection_scores:0,detection_boxes:0
Обновление cmake:
wget https://www.cmake.org/files/v3.18/cmake-3.18.1.tar.gz  
tar xpvf cmake-3.18.1.tar.gz cmake-3.18.1/
cd cmake-3.18.1/
sudo apt-get install curl  
sudo apt-get install libssl-dev libcur14-openssl-dev
./bootstrap --system-curl
sudo make
sudo make install
export CMAKE_ROOT=/usr/local/share/cmake-3.18
. ~/.bashrc 

# echo 'export PATH=/home/varya/cmake-3.18.1/bin/:$PATH' >> ~/.bashrc
# source ~/.bashrc
Устанавливаем onnx-tensorrt:
git clone --recurse-submodules https://github.com/onnx/onnx-tensorrt.git
cd /home/varya/onnx-tensorrt/
mkdir build
cd build/
cmake .. -DCUDA_INCLUDE_DIRS=/usr/local/cuda/include -DTENSORRT_ROOT=/usr/src/tensorrt -DGPU_ARCHS="53"    
make -j4
sudo make install
Конвертирование графа .onnx в trt:
onnx2trt /home/varya/frozen_ssd.onnx -o /home/varya/frozen_ssd_onnx.trt
